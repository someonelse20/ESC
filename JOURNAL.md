# 4/1/2026 4 PM - Wrote Firmware

_Time spent: 7h_

I wrote all the firmware to get the esc running. At first I was looking at implementing a field oriented controll algorithm but this would drive all three phases at once and not leave any floating to measure the back emf voltage. From what I could find the back emf can still be found by measuring the current but I couldn't find out how. I settled on trapezoidal controll which is simpler and would always leave a phase floating to measure the back emf. To control the speed I used a pid library I wrote a while ago. The actual algorithm is pretty simple and my main loop is only about 12 lines long. I'm not even using the rotor angle and will probably remove it eventualy. 
![Screenshot 2026-04-01 124457](https://stasis.hackclub-assets.com/images/1775061972528-zc3cpv.png)
Since I decided to not use field oriented control I don't need to measure the current but I still wrote a function to do so. After looking at the datasheet for the gate driver I realized that all the default settings are fine for what I need so I don't need to confgure it at startup. I have a half finished header file for a driver I was going to write for this. 

One quick side project I had was to resize all of the small vias to make them large enough so I don't get an extra fee from jlcpcb. I also increased the track width for the mosfet gate traces.

# 3/23/2026 3 PM - Finished Most of Schematic

_Time spent: 6h_

To finish the schematic I had to do a lot of background research to know what is actually hapenning. One of the more important parts is sensing the back emf signal to determine the position of the rotor to know when to switch to the next step in the 6 step sequence. Conveniently when the back emf (bemf) voltage crosses zero is the most efficient time to switch to the next step. Unconveniently the voltage is refrenced to the ground in the motor which we do not know. This can be solved by delaying switching steps by about 30 degrees of the motor or simulating the virtual ground. I wasn't sure what to do so I added the hardware for the virtual ground in case I wanted it.

# 3/19/2026 10 PM - Selected Components and Started Schematic

_Time spent: 4h_

Designing a custom ESC looked like a cool chalange so I started researching and I found the DRV8323R gate driver from TI. I chose this one because it has a SPI interface so its settings can be changed on the fly and it has an integrated buck regulator so I won't have to add that too. I then chose the STM32G431CBT6 because looking online it seemed that the STM32 G4 series was made for things like this and this specific model seems to be popular on JLCPCB. I then wired up the external mosfets according to the gate driver datasheet. 
![2026-03-19-184207_hyprshot](https://stasis.hackclub-assets.com/images/1773960443072-uv4jec.png)
I then added the power source pins and added a TVS diode to protect from overvoltage and a p mosfet for reverse voltage protection. I then added an inductor and a bunch of capacitors to smooth the input supply. Later I want to find out if the inductor is strictly needed because it's really big and I want to make this as small as I can.
